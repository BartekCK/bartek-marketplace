# Project Configuration Reference

The `.serena/project.yml` file controls all project-level Serena settings. It is generated by `serena project create` and can be edited manually.

## Core Settings

### `project_name`
The name by which the project can be referenced within Serena.
```yaml
project_name: "my-project"
```

### `languages`
List of languages for which language servers are started. The first language is the default and its server acts as fallback.
```yaml
languages:
- python
- typescript
```

### `encoding`
Text file encoding. Default: `utf-8`.
```yaml
encoding: "utf-8"
```

### `language_backend`
The language backend to use. Valid values: `LSP`, `JetBrains`. If not set, the global setting from `serena_config.yml` is used. Fixed at startup ŌĆö cannot be changed mid-session.
```yaml
language_backend: LSP
```

## File Filtering

### `ignore_all_files_in_gitignore`
Whether to use the project's `.gitignore` files. Default: `true`.
```yaml
ignore_all_files_in_gitignore: true
```

### `ignored_paths`
Additional paths to ignore (gitignore syntax with `*` and `**` support). Applied in addition to global `ignored_paths` from `serena_config.yml`.
```yaml
ignored_paths:
- "node_modules/**"
- "dist/**"
```

## Access Control

### `read_only`
Disable all editing tools. Default: `false`.
```yaml
read_only: false
```

## Tool Configuration

### `excluded_tools`
List of tool names to exclude from the available set.
```yaml
excluded_tools:
- execute_shell_command
```

### `included_optional_tools`
List of optional tools to include that would otherwise be disabled.
```yaml
included_optional_tools: []
```

### `fixed_tools`
Fixed set of tools to use as the base tool set, replacing defaults. Cannot be combined with non-empty `excluded_tools` or `included_optional_tools`.
```yaml
fixed_tools: []
```

## Modes

### `base_modes`
Modes always included in the active set. Overrides global config if defined.
```yaml
base_modes:
- editing
```

### `default_modes`
Modes activated by default. Can be overridden by CLI `--mode` parameters.
```yaml
default_modes:
- interactive
```

## Prompt and Indexing

### `initial_prompt`
Prompt given to the LLM upon activating the project (always loaded, unlike memories).
```yaml
initial_prompt: "This is a Django web application."
```

### `symbol_info_budget`
Time budget (seconds) per tool call for retrieving additional symbol information. Overrides global config.
```yaml
symbol_info_budget: 2.0
```

## Available Tools Reference

Key tools available after activation:

- **`activate_project`** ŌĆö Activate a project by name or path
- **`find_symbol`** ŌĆö Global/local symbol search with filtering
- **`get_symbols_overview`** ŌĆö Top-level symbols in a file
- **`find_referencing_symbols`** ŌĆö Find symbols referencing a given symbol
- **`replace_symbol_body`** ŌĆö Replace a symbol's full definition
- **`insert_before_symbol`** / **`insert_after_symbol`** ŌĆö Insert code around symbols
- **`search_for_pattern`** ŌĆö Regex pattern search across the project
- **`read_file`** / **`create_text_file`** ŌĆö File I/O
- **`list_dir`** ŌĆö Directory listing
- **`execute_shell_command`** ŌĆö Run shell commands
- **`onboarding`** ŌĆö Analyze project structure
- **`write_memory`** / **`read_memory`** / **`list_memories`** ŌĆö Persistent project notes
